# Introduction

Use a funnel adapted to fluid mechanics:

1. Establish the physical or engineering problem and controlling regimes.
2. Introduce the relevant model hierarchy and why common approaches struggle.
3. Group prior work by approach, assumption, or limitation.
4. State the unresolved gap precisely.
5. Present the contribution as a set of verifiable advances, not a list of paper sections.
6. Briefly identify the evidence used to support them.

For comparison papers, introduce both methods under a common problem before naming their difference. For reviews, define scope and organizing taxonomy instead of manufacturing a single-paper gap.

Avoid textbook histories, citation catalogues, application lists with no argumentative role, and detailed derivations. A numerical limitation should name its source, such as scale separation, stiffness, mesh constraint, numerical dissipation, force imbalance, conservation error, geometric complexity, data scarcity, or computational cost.

If the introduction is a list of citations, regroup it by limitation source or
model hierarchy. If the contribution list is vague, convert each item into
`what changes -> what property or insight it enables -> what evidence will test
it`.
