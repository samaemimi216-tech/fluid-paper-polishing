# Results and discussion

Structure each result unit as:

`question -> conditions -> observation -> quantitative comparison -> physical or numerical explanation -> boundary`

- Report flow structures with named quantities, locations, times, and regimes.
- Describe trends before mechanisms: `increases`, `decreases`, `shifts`, `broadens`, `oscillates`, `approaches`, `deviates`.
- Use `because` only when causality is supported; otherwise use `is consistent with`, `can be attributed to`, or `may arise from`.
- Tie every figure to a scientific question; do not narrate curves from left to right.
- Distinguish physical effects from numerical artifacts and parameter effects from regime changes.
- In a combined section, keep the local order `result first -> interpretation second`; do not merge them inside one ambiguous sentence.
- Discuss failure cases, sensitivity, and applicability limits alongside advantages.
