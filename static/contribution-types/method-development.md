# Contribution type: method development

Build the argument as:

`specific limitation -> design change -> property created -> analysis or verification -> performance under target conditions -> applicability boundary`

- Name the numerical or modeling defect before presenting the remedy.
- Explain the operative change at the flux, collision, reconstruction, closure, boundary, coupling, or implementation level.
- Separate a property guaranteed by construction from one observed in tests.
- Support each advertised property with its own evidence; benchmark agreement alone does not establish conservation, stability, asymptotic preservation, or efficiency.
- End with the regimes and costs actually tested, not universal superiority.
