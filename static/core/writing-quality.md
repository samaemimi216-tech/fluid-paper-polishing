# Writing-quality pass

Run this after the scientific structure is repaired and before delivering prose.

- Remove throat-clearing openers such as `It is worth noting`, `In this paper we aim to`, and `With the rapid development of`.
- Replace generic intensifiers with measurable claims: `excellent`, `remarkable`, `significant`, `robust`, `superior`, `promising`, and `efficient` need a metric, baseline, uncertainty, or tested range.
- Avoid AI-typical academic filler: `delve into`, `pivotal`, `landscape`, `multifaceted`, `underscore`, `leverage`, `paradigm`, `cutting-edge`, and `comprehensive` unless the word is the precise technical choice.
- Prefer stable technical repetition over synonym cycling. Do not alternate `method`, `scheme`, `model`, `solver`, and `framework` for the same object unless the distinction is real.
- Vary sentence length by rhetorical function: compact sentences for main findings, longer sentences for conditions, caveats, and mechanisms. Methods paragraphs may be more uniform than Discussion paragraphs.
- Limit em dashes and semicolon chains. Use commas, parentheses, colons, or separate sentences when they improve readability.
- Keep equations, symbols, citation keys, figure labels, table labels, and numerical values untouched unless the user explicitly asks for technical correction.
