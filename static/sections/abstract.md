# Abstract

Choose the move order that matches the contribution type; do not force one template onto every fluid paper.

Default method-paper order:

`problem/defect -> proposed change -> operative principle -> property -> verification/comparison -> quantitative result -> scope`

Default physical-paper order:

`phenomenon/question -> setup and regime -> principal observation -> mechanism or model -> quantitative boundary -> implication`

- Put the main limitation and contribution within the opening third.
- Attach numbers to a comparison, condition, metric, and reference.
- Distinguish `recovers`, `preserves`, `agrees`, `predicts`, and `reveals`.
- Avoid benchmark inventories; name only tests essential to the claim.
- Do not end with `has promising potential`; state demonstrated use or a bounded implication.
- For method abstracts, separate the proven/discrete property from empirical
  benchmark performance.
- For data-driven abstracts, name the training data, held-out test, physical
  constraint, and deployment or solver-coupling boundary when available.
